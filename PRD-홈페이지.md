# PRD: 시스템 에어컨 · 인테리어 전문 회사 홈페이지

> 이 문서는 Claude Code로 개발하기 위한 제품 요구사항 문서(PRD)입니다.
> `[대괄호]` 안의 내용은 실제 회사 정보로 교체해 주세요.

---

## 1. 프로젝트 개요

### 1.1 목적
시스템 에어컨 설치/시공 및 인테리어 전문 회사 **[회사명]**의 공식 홈페이지 제작.
잠재 고객(가정, 상가, 사무실)이 회사를 신뢰하고 **견적 문의로 전환**되도록 하는 것이 최우선 목표.

### 1.2 핵심 목표 (우선순위 순)
1. **견적 문의 전환**: 모든 페이지에서 전화/문의로 쉽게 이어지는 동선
2. **신뢰 구축**: 시공 사례(포트폴리오), 회사 소개, 고객 후기로 전문성 어필
3. **검색 노출**: 네이버/구글 지역 검색 대응 SEO ("○○구 시스템에어컨", "○○ 인테리어" 등)
4. **모바일 최적화**: 방문자 대부분이 모바일 → 모바일 퍼스트 디자인

### 1.3 타겟 사용자
- 신축/이사 후 시스템 에어컨 설치를 알아보는 30~50대 가정 고객
- 상가/사무실 오픈을 준비하며 인테리어+냉난방 일괄 시공을 원하는 자영업자/법인
- 기존 에어컨 교체, 이전 설치, 청소/유지보수를 원하는 고객

## 2. 기술 스택

| 구분 | 선택 | 비고 |
|------|------|------|
| 프레임워크 | **Next.js 15 (App Router)** | React 19, Server Components 기본 |
| 언어 | TypeScript | strict 모드 |
| 스타일링 | Tailwind CSS v4 | shadcn/ui 컴포넌트 활용 가능 |
| 폼 처리 | React Hook Form + Zod | 견적 문의 폼 검증 |
| 이메일 발송 | Resend (또는 Nodemailer + SMTP) | 문의 접수 알림 |
| 이미지 | next/image | 시공 사진 최적화 필수 |
| 애니메이션 | Framer Motion (선택) | 과하지 않게, 스크롤 리빌 정도 |
| 배포 | Vercel | 커스텀 도메인 연결 |
| 분석 | Google Analytics 4 + 네이버 서치어드바이저 | |

**콘텐츠 관리 방식 (택 1, 기본은 A):**
- **A. 파일 기반 (권장 초기안)**: 시공 사례를 MDX 또는 JSON + 이미지 폴더로 관리. DB 불필요, 유지비 0원.
- B. 헤드리스 CMS (Sanity 등): 사장님이 직접 사례를 올려야 하는 경우 2차 단계에서 도입.

## 4. 페이지별 상세 요구사항

### 4.1 홈 (`/`)
가장 중요한 페이지. 위에서 아래 순서:

1. **히어로 섹션**
   - 대표 시공 사진 배경 (고품질)
   - 헤드라인: "[지역] 시스템 에어컨 · 인테리어 전문 시공" 류의 지역+서비스 키워드 포함
   - CTA 버튼 2개: `무료 견적 문의` (primary) / `시공 사례 보기` (secondary)
   - 모바일에서 전화 바로걸기 버튼 노출
2. **핵심 강점 3~4개** (아이콘 + 짧은 문구)
   - 예: 경력 [N]년, 직영 시공, A/S 보장, 무료 현장 실측
3. **서비스 요약** — 에어컨 / 인테리어 두 카드, 각 상세 페이지로 링크
4. **대표 시공 사례 4~6개** — 포트폴리오에서 추천 항목, 더보기 링크
5. **시공 프로세스** — 상담 → 실측 → 견적 → 시공 → A/S (스텝 UI)
6. **고객 후기** — 3~5개 슬라이드/카드 (텍스트 + 별점)
7. **취급 브랜드 로고** — LG, 삼성, 캐리어 등 (텍스트로만, 로고 이미지는 사용권 확인 후)
8. **하단 CTA 배너** — "지금 무료 견적 받아보세요" + 전화번호 크게

### 4.2 회사 소개 (`/about`)
- 인사말, 회사 연혁, 사업자 정보 (상호, 대표자, 사업자등록번호, 주소)
- 보유 면허/자격 (예: 기계설비공사업, 실내건축공사업 등 해당 시)
- 시공 지역 안내 (예: 서울 전 지역, 경기 일부)
- 오시는 길: 주소 + 지도 (초기엔 정적 지도 이미지 또는 네이버 지도 링크로 충분)

### 4.3 서비스 페이지 (`/services/*`)
공통 구조: 서비스 설명 → 세부 항목 → 시공 프로세스 → 관련 시공 사례 3개 → FAQ → CTA

**시스템 에어컨 (`/services/aircon`)**
- 세부 항목: 신규 설치(1way/4way 등), 이전 설치, 교체, 배관 공사, 실외기 위치 시공
- 대상 공간: 아파트/주택/상가/사무실
- "설치 비용은 어떻게 결정되나요?" 등 FAQ 5개 내외 (JSON-LD FAQ 스키마 적용)

**인테리어 (`/services/interior`)**
- 세부 항목: 주거 인테리어(전체/부분), 상업 공간(카페, 사무실, 매장), 리모델링
- 에어컨+인테리어 **일괄 시공 패키지** 강조 (차별화 포인트)

### 4.4 시공 사례 (`/portfolio`, `/portfolio/[slug]`)
- **목록**: 카드 그리드 (썸네일 + 제목 + 지역 + 카테고리 태그), 카테고리 필터 탭 (전체/에어컨/인테리어)
- **상세**:
  - 프로젝트 정보 테이블: 위치(구 단위까지만), 공간 유형, 시공 범위, 기간
  - 사진 갤러리 (before/after 비교 UI가 있으면 좋음, 없으면 일반 갤러리)
  - 시공 포인트 설명 텍스트
  - 하단: 관련 사례 + 견적 문의 CTA
- 데이터 구조 예시 (`content/portfolio/*.mdx` frontmatter):
```yaml
title: "강남구 30평 아파트 시스템에어컨 4대 설치"
slug: "gangnam-apt-aircon-2026-01"
category: "aircon"        # aircon | interior | both
location: "서울 강남구"
spaceType: "아파트 30평"
duration: "2일"
date: "2026-01-15"
thumbnail: "/images/portfolio/gangnam-apt/01.jpg"
images: [ ... ]
featured: true
```

### 4.5 견적 문의 (`/contact`)
- **폼 필드**:
  - 이름 (필수), 연락처 (필수, 휴대폰 형식 검증)
  - 문의 유형 (필수, 선택: 시스템 에어컨 / 인테리어 / 둘 다 / 기타)
  - 시공 지역 (필수), 공간 유형 (선택: 아파트/주택/상가/사무실)
  - 상세 내용 (선택, textarea)
  - 개인정보 수집·이용 동의 체크박스 (필수) — 동의 내용 펼쳐보기 제공
- **제출 처리**: Server Action → 이메일 발송 (관리자 수신) → 성공 화면 ("영업일 기준 24시간 내 연락드립니다")
- 스팸 방지: honeypot 필드 + rate limiting (간단하게)
- 폼 옆/아래에 **직접 연락 수단** 병기: 전화번호 (tel: 링크), 카카오톡 채널 링크 `[채널 URL]`
- 실패 시 에러 메시지와 전화번호 안내 (문의가 유실되면 안 됨)

### 4.6 공통 레이아웃
- **헤더**: 로고 / GNB (소개, 서비스, 시공사례, 문의) / 우측 `견적 문의` 버튼. 모바일 햄버거 메뉴.
- **모바일 하단 고정 바**: `전화하기` + `견적 문의` 버튼 (전환율에 매우 중요)
- **푸터**: 사업자 정보, 연락처, 시공 지역, 개인정보처리방침 링크, ⓒ 표기
- **플로팅 버튼** (선택): 카카오톡 채널 상담

## 5. 비기능 요구사항

### 5.1 SEO (중요)
- 페이지별 `metadata` (title, description) — 지역+서비스 키워드 포함
  - 예: `서울 [지역] 시스템에어컨 설치 · 인테리어 전문 | [회사명]`
- `sitemap.xml`, `robots.txt` 자동 생성 (next-sitemap 또는 App Router 내장 방식)
- 구조화 데이터 (JSON-LD): `LocalBusiness` (홈), `FAQPage` (서비스), `BreadcrumbList`
- Open Graph 이미지 (대표 시공 사진 기반)
- 네이버 서치어드바이저 + 구글 서치콘솔 소유 확인 메타태그 자리 확보
- 시공 사례 페이지가 롱테일 검색 유입 채널 → slug/제목에 지역·평수·서비스 키워드 자연스럽게 포함

### 5.2 성능
- Lighthouse 모바일 90+ 목표
- 모든 시공 사진 next/image + WebP/AVIF, 히어로 이미지 priority
- 폰트: Pretendard (subset, `next/font/local` 또는 CDN)

### 5.3 접근성 · 기타
- 시맨틱 마크업, 이미지 alt 텍스트 (시공 내용 설명)
- 반응형 브레이크포인트: 모바일 우선 → sm/md/lg
- 개인정보처리방침 페이지 필수 (문의 폼에서 개인정보 수집하므로)

## 6. 디자인 가이드

- **톤**: 깔끔하고 신뢰감 있는 시공 전문 업체 느낌. 화려한 애니메이션보다 **사진 품질**이 핵심.
- **컬러**: 화이트 베이스 + 포인트 컬러 1개 (예: 딥 네이비 `#1B3A5C` 또는 시원한 블루 계열 — 에어컨 연상)
- **타이포**: Pretendard. 제목은 굵고 크게, 본문은 가독성 위주.
- 시공 사진은 일관된 비율(4:3 또는 3:2)로 크롭
- 참고 레퍼런스 사이트: `[있으면 URL 기재]`

## 7. 개발 단계 (Claude Code 작업 순서 제안)

**Phase 1 — 기반 (MVP)**
1. Next.js 15 + TypeScript + Tailwind 프로젝트 셋업
2. 공통 레이아웃 (헤더/푸터/모바일 하단 바)
3. 홈 페이지 전체 섹션 (더미 이미지/텍스트로)
4. 견적 문의 폼 + Server Action + 이메일 발송
5. 개인정보처리방침 페이지

**Phase 2 — 콘텐츠**
6. 서비스 페이지 2종 (에어컨/인테리어) + FAQ
7. 시공 사례 목록/상세 (MDX 기반) + 필터
8. 회사 소개 페이지

**Phase 3 — 마무리**
9. SEO (metadata, sitemap, JSON-LD, OG 이미지)
10. 성능 최적화, 반응형 QA
11. Vercel 배포 + 도메인 연결 + GA4/서치콘솔 등록
